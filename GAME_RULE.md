## 💡 게임 개발 팁

### Teachable Machine 모델 학습 시 주의사항

1. **다양한 환경에서 샘플 수집**
   - 밝은 곳 / 어두운 곳
   - 다양한 배경
   - 여러 각도와 거리

2. **충분한 샘플 수집**
   - 각 클래스당 최소 50개 이상 권장
   - 더 많을수록 정확도 향상

3. **대기 상태 클래스 추가**
   - "정면" 또는 "대기" 클래스를 만들어 아무 포즈도 취하지 않은 상태를 학습
   - 오인식 방지에 도움

4. **⚠️ 모델 업데이트 후 반드시 확인**
   - 모델 학습 완료 후 `my_model/metadata.json` 파일 확인 필수
   - 파일을 열어서 `"labels"` 항목에 학습한 클래스 이름이 정확히 들어있는지 확인
   - 예시:
   ```json
   {
     "labels": ["왼쪽", "오른쪽", "정면", "위", "아래"],
     ...
   }
   ```
   - **위 라벨 정보를 기준으로 게임 코드를 작성해야 함**
   - 라벨 이름이 GAME_RULE.md의 "필요한 포즈 목록"과 일치하는지 확인

5. **⚠️ 라벨 불일치 시 대응 방법**

   만약 metadata.json의 라벨과 GAME_RULE.md의 포즈 목록이 다른 경우:

   **옵션 A: metadata.json 기준으로 GAME_RULE.md 수정 (권장)**
   - metadata.json의 실제 라벨 이름에 맞춰 GAME_RULE.md의 "필요한 포즈 목록" 수정
   - 게임 코드 작성 시 정확한 라벨 사용 가능

   **옵션 B: 라벨 매핑 테이블 작성**
   - 두 이름이 다를 경우, 매핑 관계를 명확히 정의
   - GAME_RULE.md에 다음과 같이 매핑 테이블 추가:

   ```markdown
   ## 라벨 매핑 (metadata.json ↔ 게임 표시 이름)

   | metadata.json 라벨 | 게임에서 표시할 이름 | 설명 |
   |-------------------|-------------------|------|
   | 왼쪽 | Left | 왼쪽을 보세요 |
   | 오른쪽 | Right | 오른쪽을 보세요 |
   | 정면 | Center | 정면을 보세요 |
   | 위 | Up | 위를 보세요 |
   | 아래 | Down | 아래를 보세요 |
   ```

   - AI 코딩 도구에게 "metadata.json의 라벨을 사용하되,
     게임 UI에는 매핑 테이블의 '게임에서 표시할 이름'을 사용해줘"라고 요청

### gameEngine.js 커스터마이징 가이드

게임 로직을 구현할 때 다음 사항을 참고하세요:

```javascript
});
```
